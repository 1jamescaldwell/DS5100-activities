Copy the file student.py from the course repo to your private repo.

Add a new method to student.py called unenroll_in_course().

The method should raise an exception if the student is not enrolled in the course.

Then write a unit test file student_test.py with unittest methods that do the following:

  • test_01 tests the incrementer works.
  • test_02 tests a course is actually removed with the new method.
  • test_03 tests to see if the new method does not increment courses if the student was not enrolled in the course to begin with.

Note that we are using num_courses as a counter when we could have used len(set(self.courses)) to get the number of courses. This is just for demonstration purposes.

Run the test script and see your results.

class Student:
    
    def __init__(self, name, courses=None):
        self.name = name # string type
        self.courses = [] if courses is None else courses # list of strings
        self.num_courses = len(self.courses)
        
    def enroll_in_course(self, course_name):
        "Enrolls the student in a course."
        self.courses.append(course_name)
        self.num_courses += 1 # increment the number of courses
    
    def unenroll_in_course(self, course_name):
        "Unenrolls the student from a course."
        if course_name in self.courses:
            self.courses.remove(course_name)
            self.num_courses -= 1 # decrememt the number of courses
        else:
            # print("Student was not enrolled in that course")
            # raise Exception("Student was not enrolled in that course")
            raise ValueError("Date provided can't be in the past")

Exercise 8.5

Create a class called Car and a class that inherits from this class called Subaru.

For the Car class, add the following attributes and methods: - body_type which takes the values suv, crossover, sedan, hatchback, sport - engine_type: which takes the values electric, gas, diesel, hybrid - __init__(): which takes values for the two attributes - describe_car(): which returns a string with the body and engine types as f"{engine_type} {body_type}"

The object should raise a ValueError exception if either of the values passed to the attributes are not in the specified lists of values.

For the Subaru class, add the folliwing: - year: the four-number year of manufacture, e.g. \(2023\) - model: the specific model of the car, e.g. “outback” - trim: the special trim of the car, e.g. “sport” - __init__(): which takes values for the previous attributes and the Car attributes. - describe(): which returns a string with the year, model, and trim, and with the Car description in parentheses.

Then create a new Subaru object and initialize it with the following values: - body_type = 'crossover' - engine_type = 'gas' - year = '2022' - model = 'outback' - trim = 'sport'

Then call the describe method.

class Car:
    
    body_types = ["suv", "crossover", "sedan", "hatchback", "sport"]
    engine_types = ["electric", "gas", "diesel", "hybrid"]
    
    def __init__(self, body_type, engine_type):
        if body_type not in self.body_types:
            raise ValueError("Invalid body type.")
        if engine_type not in self.engine_types:
            raise ValueError("Invalid engine type.")
        self.body_type = body_type
        self.engine_type = engine_type
        
    def describe_car(self):
        return f"{self.engine_type} {self.body_type}"
        
class Subaru(Car):
    
    def __init__(self, body_type, engine_type, year, model, trim=None):
        self.year = year
        self.model = model
        self.trim = trim
        Car.__init__(self, body_type, engine_type)
        # super().__init__(body_type, engine_type) # Another option

    def describe(self):
        return f"{self.year} {self.model} {self.trim} ({self.describe_car()})"
